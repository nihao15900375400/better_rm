# Del

## 项目简介
`Del` 是一个基于 Rust 开发的安全文件删除与回收站管理工具。该程序旨在提供比系统原生 `rm` 命令更安全的文件删除方案，通过将目标文件/目录压缩归档至指定回收站路径，并记录元数据至 SQLite 数据库，实现误删恢复、条件查询、安全清空与永久删除等功能。

## 快速开始

```bash
# 0. 下载并执行rust安装脚本，克隆并编译
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env

git clone https://gitee.com/ywnh1/rust_del.git
cd rust_del

cargo build --release
cp target/release/del /bin
```

```bash
# 1. 初始化配置文件与数据库（首次运行必需）
del --init

# 2. 将文件或目录移入回收站
del file.txt directory/

# 3. 查看回收站已删除的文件列表
del --list

# 4. 恢复最近删除的文件（或指定ID）
del --undo
del --undo 1

# 5. 根据名称模糊搜索回收站文件
del --select "%.log"
```

## 命令行参数详解
| 参数 | 简写 | 说明 | 示例 |
|:---|:---|:---|:---|
| `<path>` | 无 | 指定要删除的文件或目录路径（支持多路径） | `del a.txt b/` |
| `--init` | `-i` | 初始化回收站目录结构与配置数据库。若文件已存在会提示是否覆盖。 | `del -i` |
| `--undo` | `-u` | 从回收站恢复文件。若不跟参数，默认恢复最近一次删除的文件；支持传入一个或多个回收站 `id`。恢复前需确保原路径无同名文件。 | `del -u 3` |
| `--delete` | `-d` | 根据 `id` 永久删除回收站中的记录及对应归档文件。 | `del -d 1 2` |
| `--recurse` | `-r` | 递归操作。配合 `--force` 使用时等效于 `rm -r`。 | `del -rf dir/` |
| `--force` | `-f` | 绕过回收站机制，直接调用系统 `rm` 命令永久删除。 | `del -f file.txt` |
| `--list` | `-l` | 以分页形式列出回收站中所有文件记录。 | `del -l` |
| `--select` | `-s` | 按文件名进行模糊匹配查询。 | `del -s "report_%.doc"` |
| `--select-from` | 无 | 按指定数据库字段进行模糊筛选。支持字段：`name`, `id`, `time`, `original-path`, `size`。 | `del --select-from time "2024-05-%"` |
| `--config` | `-c` | 尝试使用 `nano` 编辑器打开配置文件；若未安装 `nano` 则打印配置路径。 | `del -c` |
| `--empty` | `-e` | 清空回收站（删除所有归档文件并重置数据库）。需交互式输入 `Y` 确认。 | `del -e` |

## 配置文件说明
配置文件默认位于 `~/.config/del/config.json`。可通过 `del --config` 查看或编辑。

```json
{
  "trash": "~/.del_trash",
  "archive_tool": "gz",
  "disable_list": [
    "/", "/home", "/root", "/etc", "/usr",
    "/var", "/boot", "/lib", "/lib64",
    "/proc", "/sys", "/dev", "/tmp",
    ".", "..", "~"
  ]
}
```
- **`trash`**: 回收站物理存储路径。建议使用绝对路径。
- **`archive_tool`**: 归档压缩算法，可选值：`xz2`, `zstd`, `bz2`, `gz`。
- **`disable_list`**: 路径保护黑名单。程序在执行删除前会进行精确路径匹配，若目标路径在列表中则拒绝操作并退出，防止误删系统关键目录。

## 高级查询与筛选语法
`--select` 与 `--select-from` 底层基于 SQLite 的 `LIKE` 操作符，支持以下通配规则：
- `%`：匹配任意长度的字符序列（包含零个字符）。
- `_`：匹配任意单个字符。
- `\`：转义字符。用于将 `%` 或 `_` 作为普通字符匹配。

**查询示例**
```bash
# 查找所有以 .txt 结尾的记录
del --select "%.txt"

# 查找 2024 年 12 月任意日期的删除记录
del --select-from time "2024-12-%"

# 查找原始路径中包含下划线的记录
del --select-from original-path "%\_%"
```

## 注意事项与安全机制
1. **路径解析规则**：程序自动处理 `~` 符号与相对路径，统一转换为绝对路径进行匹配与归档。
2. **恢复限制**：执行 `--undo` 时，程序会校验原路径是否已存在同名文件或目录。若存在则中止恢复并报错，避免覆盖现有数据。
3. **强制删除风险**：使用 `--force` 参数将直接调用系统 `rm` 命令，数据**不可恢复**。请务必谨慎使用。
4. **数据库依赖**：回收状态与元数据由 `~/.config/del/trash.db` 管理。若手动删除 `.bak` 归档文件但未清理数据库，列表显示与实际存储将不一致。建议使用程序内置命令管理。
5. **压缩归档命名**：回收站内的文件统一命名为 `<时间戳>.bak`，时间格式为 `YYYY-MM-DD_HH:MM:SS`。

## 总结
`Del` 从**安全防护、操作灵活性与数据可追溯性**三个维度构建了完整的文件删除工作流：
1. **安全防护**：通过 `disable_list` 硬编码保护系统关键路径，默认拦截根目录、用户目录及核心系统目录的误删风险。
2. **操作灵活性**：提供多种压缩算法选择（xz2/zstd/bz2/gz）、分页查看、按字段模糊查询与交互式确认机制，适配不同场景下的运维与日常使用需求。
3. **数据可追溯性**：基于 SQLite 记录文件原始路径、归档路径、大小、时间与压缩类型，确保删除操作可查、可逆，并支持按需永久清理。

该工具适合作为 Linux/macOS 环境下 `rm` 命令的安全替代方案。日常使用建议保持 `--init` 初始化后的默认配置，定期通过 `--list` 与 `--select` 巡检回收站状态，并使用 `--empty` 清理过期数据以释放磁盘空间。
